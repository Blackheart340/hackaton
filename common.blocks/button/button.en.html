button

button block is used to manage a size, state, content and appearance of a button.

Brief overview

Modifiers of the block

Modifier Available values Use cases Description
type ‘link’, ‘submit’ BEMJSON A button type.
view ‘action’, ‘pseudo’ BEMJSON Visual highlighting.
togglable ‘check’, ‘radio’ BEMJSON A toggle type of a button.
pressed true Automatic A pressed state.
disabled true BEMJSON, JS A disabled state.
focused true BEMJSON, JS The block is in focus.
size ‘s’, ‘m’, ‘l’, ‘xl’ BEMJSON A button size. The sizes are used only for a button with theme modifier with islands value.
theme ‘islands’ BEMJSON A custom design.

Custom fields of the block

Field Type description
name String A value that will be sent to a server or obtained using client scripts. Do not use for a button with link behavior.
val String A value that will be sent to a server or obtained using client scripts. Do not use for a button with link behavior.
text String A button text.
url String A link address. Use only for a button with link behavior.
icon BEMJSON An icon on a button that is based on icon block.
title String A tooltip.
id BEMJSON A unique identifier of a button.
tabIndex Number A tab order between buttons.

Block overview

Modifiers of the block

type modifier

Available values: 'link', 'submit'.

Use case: BEMJSON.

A button with link behavior (type modifier with link value)

Use type modifier with link value to create a button that behaves like a link.

Specify additional a link address in url field in BEMJSON input data.

{
    block : 'button',
    url : 'www.bem.info',
    text : 'Try to use BEM!',
    mods : { theme : 'islands', size : 'm', type : 'link' }
}
A form submit button (type modifier with submit value)

Use type modifier with submit value to create a button to send data to a server. This button type must be located inside a form.

{
    block : 'button',
    text : 'To send data',
    mods : { theme : 'islands', size : 'm', type : 'submit' }
}

view modifier

Available values: 'action', 'pseudo'.

Use case: BEMJSON.

An action button (view modifier with action value)

Use view modifier with action value to visually highlight a button on a page. Could be used as a promo button.

{
    block : 'button',
    mods : { theme : 'islands', size : 'm', view : 'action' },
    text : 'Buy now!'
}
A pseudobutton (view modifier with pseudo value)

Use view modifier with pseudo value to change visual representation of a button. For example in case when you do not need to focus attention on the button.

{
    block : 'button',
    mods : { theme : 'islands', size : 'm', view : 'pseudo' },
    text : 'With transparent background'
}

{
    block : 'button',
    mods : { theme : 'islands', size : 'm', view : 'pseudo', disabled : true },
    text : 'Without boarders'
}

togglable modifier

Available values: 'check', 'radio'.

Use case: BEMJSON.

Defines behavior of a pressed button.

A toggle button (togglable modifier with check value)

The first click presses the button, and the second releases it.

{
    block : 'button',
    text : 'Pressed button',
    mods : { theme : 'islands', size : 'm', togglable : 'check' }
}
A radio-button (togglable modifier with radio value)

The first click presses the button and it cannot be released by the next click.
This button type is used as a part of radio-group.

{
    block : 'button',
    text : 'Selected button',
    mods : { theme : 'islands', size : 'm', togglable : 'radio' }
}

An example of button usage as a part of a radio-group:

{
    block : 'radio-group',
    mods : { theme : 'islands', size : 'm', type : 'button' },
    name : 'radio-button',
    options : [
        { val : 1, text : 'First' },
        { val : 2, text : 'Second', checked : true },
        { val : 3, text : 'Third' }
    ]
}

pressed modifier

Available value: true.

Use case: automatic.

pressed modifier with true value is added to a button automatically at the moment when this button is pressed.

This modifier is used for buttons with togglable modifier.

disabled modifier

Available value: true.

Use case: BEMJSON, JS.

The modifier provides inactive/disabled state to the block. disabled modifier is used to make the block visible but not available for user actions.

{
    block : 'button',
    text : 'Disabled',
    mods : { theme : 'islands', size : 'm', disabled : true }
}

focused modifier

Available value: true.

Use case: BEMJSON, JS.

The modifier provides focus to the block.

{
    block : 'button',
    text : 'In focus',
    mods : { theme : 'islands', size : 'm', focused : true }
}

size modifier

Available values for islands theme: 's', 'm', 'l', 'xl'.

Use case: BEMJSON.

Use size modifier only for blocks with islands theme.

Provides all types of buttons with size value.

s

{
    block : 'button',
    text : 'Size s',
    mods : { theme : 'islands', size : 's' }
}

m

{
    block : 'button',
    text : 'Size m',
    mods : { theme : 'islands', size : 'm' }
}

l

{
    block : 'button',
    text : 'Size l',
    mods : { theme : 'islands', size : 'l' }
}

xl

{
    block : 'button',
    text : 'Size xl',
    mods : { theme : 'islands', size : 'xl' }
}

theme modifier

Available value: 'islands'.

Use case: BEMJSON.

If theme modifier is not specified, native representation of a control is applied.

Note! Choosing a theme normal requires size modifier usage.

{
    block : 'button',
    text : 'Islands theme',
    mods : { theme : 'islands', size : 'm' }
}

Custom fields of the block

name field

Specifies a value that is sent to a server. This value always goes in pair with a value of val value.

Do not use name field for a button with link behavior.

val field

Specifies a value that is sent to a server. This value always goes in pair with a value of name value.

Do not use val field for a button with link behavior.

{
    block : 'button',
    text : 'Check the result',
    name : 'Test #1',
    val : 'Passed',
    mods : { theme : 'islands', size : 'm', type : 'submit' }
}

text field

Specifies a text on a button.

url field

Specifies a link address that will be opened by clicking a button with link behavior.

You must use this field only for a button with link behavior.

icon field

Specifies an icon on a button. Declare the icon in BEMJSON using icon block.

{
    block : 'button',
    text : 'Twitter',
    mods : { theme : 'normal', size : 'm' },
    icon : {
        block : 'icon',
        mods : { social : 'twitter' }
    }
}

title title

Specifies a tooltip content. The tooltip appearance depends on a browser and your operating system settings. You cannot change it applying HTML or different styles.

{
    block : 'button',
    text : 'Check the result',
    name : 'Test #1',
    val : 'Passed',
    title : 'This button shows the test results'
    mods : { theme : 'islands', size : 'm', type : 'submit' }
}

id field

Specifies a unique identifier of a button.

tabIndex field

Specifies a tab order between the buttons by pressing Tab.
